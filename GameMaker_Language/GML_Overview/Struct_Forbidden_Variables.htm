Struct Forbidden Variables

When creating a struct, it is possible to use certain built-in variables as member variable names, for example:

mystruct =
{
    object_index : obj_Player,
    speed : 3,
    image_blend : c_red
}

However, only local and instance scope built-in variables can be used this way and assigned values as if they were regular struct member variables. Using global scope built-in variables is forbidden and will cause issues with your game. Below you can find a full list of these variables so that you know which ones to avoid.

The following built-in global variables are all available for use in your projects but cannot be used as struct member variables:

  • argument[n]
  • argument0 ... argument15
  • argument_count
  • async_load
  • debug_mode
  • pointer_invalid
  • pointer_null
  • undefined
  • NaN
  • infinity
  • true
  • false
  • pi
  • room
  • room_first
  • room_last
  • room_width
  • room_height
  • room_persistent
  • score
  • lives
  • health
  • game_id
  • game_display_name
  • game_project_name
  • game_save_id
  • working_directory
  • temp_directory
  • program_directory
  • instance_count
  • instance_id
  • view_enabled
  • view_current
  • view_visible
  • view_xport
  • view_yport
  • view_wport
  • view_hport
  • view_surface_id
  • view_camera
  • mouse_x
  • mouse_y
  • mouse_button
  • mouse_lastbutton
  • keyboard_key
  • keyboard_lastkey
  • keyboard_lastchar
  • keyboard_string
  • cursor_sprite
  • fps
  • fps_real
  • current_time
  • current_year
  • current_month
  • current_day
  • current_weekday
  • current_hour
  • current_minute
  • current_second
  • event_type
  • event_number
  • event_object
  • event_action
  • application_surface
  • font_texture_page_size
  • os_type
  • os_device
  • os_version
  • os_browser
  • browser_width
  • browser_height

 

Apart from those global variables, there are also a number of obsolete built-in global variables that are unavailable due to the possibility of them being included in imported legacy products (GameMaker needs to reserve these variables so it can recognise them on import and flag them to be updated/removed):

  • gamemaker_registered
  • error_occurred
  • error_last
  • show_score
  • show_lives
  • show_health
  • caption_score
  • caption_lives
  • caption_health
  • argument_relative
  • room_caption
  • room_speed
  • background_color / background_colour
  • background_showcolor / background_showcolour
